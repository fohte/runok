## Highlights

### Breaking: `*` inside quotes is now treated as a glob pattern

Previously, `"*"` in a pattern was treated as a literal asterisk. Starting with v0.2.0, `*` inside quotes is treated as a glob wildcard, matching the same way as an unquoted `*`. To match a literal `*` character, use the `\*` escape sequence.

**What should I do?**

If your rules use `"*"` intending to match a literal asterisk, update them to use `\*`:

```yaml
# Before (v0.1.x): matched a literal * character
rules:
  - allow: 'echo "*"'

# After (v0.2.0): use \* to match a literal * character
rules:
  - allow: 'echo \*'
```

If your rules use `"*"` intending it as a wildcard (matching any arguments), no changes are needed -- it now works as you'd expect.

([#157](https://github.com/fohte/runok/pull/157))

### `runok test` subcommand

A new `runok test` subcommand lets you write test cases for your rules directly in `runok.yml`. Define expected outcomes for specific commands and verify your rules behave as intended:

```yaml
tests:
  - command: 'git status'
    expected: allow
  - command: 'rm -rf /'
    expected: deny
```

```sh
$ runok test
```

This is useful for validating rule configurations before deploying them, especially when working with complex pattern matching or presets.

([#204](https://github.com/fohte/runok/pull/204))

### Typed variable definitions with `<var:name>` placeholder

Rules can now define typed variables using `definitions.vars` and reference them with `<var:name>` in patterns. Variables support per-value types for more precise matching:

```yaml
definitions:
  vars:
    branch:
      - main
      - develop
      - 'feature/*'

rules:
  - allow: 'git checkout <var:branch>'
```

Variables can also be used in command position ([#208](https://github.com/fohte/runok/pull/208)).

([#201](https://github.com/fohte/runok/pull/201))

### Audit subcommand

The new `runok audit` subcommand provides logging and inspection of command evaluation results. It displays results in a table format with colors, making it easy to review how rules evaluate commands across your configuration.

Use `--dir` to filter audit results by directory ([#207](https://github.com/fohte/runok/pull/207)).

([#103](https://github.com/fohte/runok/pull/103), [#200](https://github.com/fohte/runok/pull/200))

## New Features

- Add `runok update-presets` command to update cached remote presets ([#197](https://github.com/fohte/runok/pull/197))
- Support `redirects` and `pipe` variables in `when` clauses for conditional rule evaluation based on shell redirections and pipes ([#203](https://github.com/fohte/runok/pull/203))
- Support fused short flag pattern matching (e.g., `-n3` matching `-n 3`) ([#205](https://github.com/fohte/runok/pull/205))

## Bug Fixes

- Reject unknown flags before `--` in `exec`/`check` subcommands instead of silently ignoring them ([#202](https://github.com/fohte/runok/pull/202))
- Remove `exec --dry-run` option (superseded by `runok check`) ([#216](https://github.com/fohte/runok/pull/216))
- Fix `<cmd>` placeholder incorrectly capturing flag-starting token sequences in wrapper evaluation ([#206](https://github.com/fohte/runok/pull/206))
